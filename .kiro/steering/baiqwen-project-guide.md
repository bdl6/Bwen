# 百问项目指导原则

## 项目宗旨
这是一个用于简历展示的项目，目标是帮助用户获得实习机会，能够在面试中自信地讨论技术细节。

## 指导方式
1. **循序渐进** - 不要一次性生成所有代码，而是分步骤指导
2. **理解优先** - 确保用户理解每一步的原理和目的
3. **面试导向** - 在关键技术点提醒可能的面试问题
4. **实践为主** - 让用户自己动手，我只提供指导和建议

## 技术栈
- 前端：Next.js 14+ (App Router) + Tailwind CSS
- 后端：Next.js API Routes
- 语言：TypeScript

## 面试高频考点提醒
在涉及以下内容时，主动提醒面试可能问到的问题：
- React Hooks 使用和原理
- Next.js SSR/CSR 区别
- API Routes 实现原理
- 流式输出技术方案
- 性能优化策略
- TypeScript 类型设计
- 状态管理方案选择
- 错误处理和边界情况
- 安全性考虑（API key 保护等）

## 代码风格
- 保持代码简洁易读
- 添加必要的注释说明关键逻辑
- 使用有意义的变量命名
- 遵循 React 和 Next.js 最佳实践
